# Architectural Decision Records (ADRs)

Lightweight log of architectural choices that have more than one
defensible answer. Each ADR captures the context, the decision, and the
consequences — so that future-us (or anyone reading the repo) can
understand *why* a choice was made, not just what was implemented.

Format: `<NNNN>-<slug>.md`. Number monotonically. Never delete; if a
decision is reversed, write a new ADR that supersedes the previous one.
